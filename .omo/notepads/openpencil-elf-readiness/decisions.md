- Task 1 contract location: `packages/docs/development/hosted-operating-modes.md`, linked from `packages/docs/.vitepress/sidebars.ts` and `AGENTS.md`, is the code-facing readiness contract for hosted operating modes and canonical identities.
- Task 1 identity decision: hosted `document.id` is the caller-facing route/storage identity; hosted `room.id` is not equal to document ID and is deterministically derived as `op_room_` + first 32 lowercase unpadded base32url chars of `sha256("openpencil:hosted-room:v1:" + document.id)`.
- Task 1 migration decision: opening authenticated local `.fig` or `.pen` files must not silently upload them; hosted `document.id` is minted only through explicit hosted create/import/promote.
- Task 5 backend boundary decision: `src/app/document/io/backend.ts` owns the `DocumentBackend` interface, operation/capability policies, and typed illegal-operation errors; `local-backend.ts` is the current app backend and `hosted-backend.ts` is contract-only until hosted runtime work lands.
- Task 5 autosave decision: local autosave remains legal only for File System Access handles or Tauri paths; browser download-only docs still require explicit Save, and hosted autosave requires a canonical hosted `document.id` plus injected runtime client.

- Task 2 source-of-truth decision: canonical ELF auth client is `ch5-packages/packages/auth/elf-auth-client`, published as `@ch5me/elf-auth-client`; `firefly-cloud/packages/elf-auth-client` is deprecated/non-canonical and must not gain new consumers.
- Task 2 payload decision: authoritative public token payload identity field is `elfUserId`; public `kiloUserId` and `fireflyUserId` are rejected by the canonical schema and must be migrated at auth boundaries.
- Task 7 hosted schema decision: D1 tables are `hosted_documents`, `hosted_snapshots`, `hosted_assets`, and `hosted_document_migrations`; R2 bytes stay in `DOCUMENTS` and `ASSETS` under keys derived from hosted `document.id`, `snapshot.id`, and `asset.id` only.
- Task 7 migration decision: legal metadata transitions are `create-empty`, `import-local`, `promote-local`, and `duplicate-hosted`; missing owner/session, malformed source format/provenance, missing snapshot bytes, missing asset metadata, local-open auto-upload, and random room joins fail explicitly before valid D1 records are created.
- Task 3 contract location: `packages/docs/development/elf-auth-naming.md`, linked from the development sidebar, is the OpenPencil code-facing auth naming doctrine and rename matrix.
- Task 3 naming decision: public/external auth surfaces use ELF vocabulary (`@ch5me/elf-auth-client`, `createElfVerifier`, `ElfTokenPayload`, `elfUserId`, `/api/elf-auth/*`, `ELF_JWT`, `X-CH5-Elf-User-Id`); internal Firefly names remain legal only when explicitly implementation-private.

- Task 2 scaffold decision: `api/` sits outside the Bun workspace (`packages/*` glob) as a standalone Cloudflare Worker package with its own `package.json`, `tsconfig.json`, and `wrangler.jsonc`. Dependencies installed independently, not through workspace resolution.
- Task 2 auth seam decision: auth in `api/src/auth.ts` is a stub seam тАФ `verifyElfToken` returns a fixed `elfUserId` for local dev; real RS256+JWKS verification via `@ch5me/elf-auth-client` wires in task 6.
- Task 2 Hono decision: the Worker uses Hono for routing (matches existing `@hono/node-server` dep in root) with CORS configured for `pencil.ch5.me` and `app.openpencil.dev`; `/health` and `/` are the initial endpoints.

- Task 8 flag resolution decision: feature flags resolve exclusively from `OPENPENCIL_HOSTED_ENV` env var тАФ no hostname checks, no `window.location` sniffing, no `import.meta.env.MODE` fallbacks. Unset env тЖТ `local` with all hosted features off.
- Task 8 topology source of truth: `config/hosted-topology.json` is the machine-readable environment matrix, aligned with `.ch5/environments.yaml`. Vite env var overrides (`VITE_HOSTED_*_ENABLED`) allow per-deployment flag toggling without rebuild.
- Task 8 type location: hosted feature flag types and validation live in `@open-pencil/core/hosted` subpath export тАФ framework-agnostic, consumable by app, CLI, MCP, and Worker tests. App-specific flag resolution lives in `src/app/hosted/flags.ts`.
- Task 8 rollout order: hostedAuth тЖТ hostedDocs тЖТ hostedCollab. Collaboration is OFF in all environments until task 12 (DO room transport) lands. Preview enables auth-only for session smoke tests; staging enables auth+docs for storage validation.
- Task 8 docs location: `packages/docs/development/hosted-environment-topology.md`, linked from the development sidebar, is the code-facing environment topology and flag contract.

- Task 9 deprecation decision: `firefly-cloud/packages/elf-auth-client/` directory retained but empty (node_modules only) тАФ full removal deferred to Firefly Cloud repo cleanup. `elf-auth-topology.md` declares it deprecated/removed, and no consumers import from the unscoped path. Public API response shapes in Firefly docs (cutover runbook) use `elfUserId` per ELF naming doctrine.
