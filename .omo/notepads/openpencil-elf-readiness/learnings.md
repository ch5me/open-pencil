- Task 1 source audit: current hosted-readiness state is still local-first. `src/router.ts` exposes `/share/:roomId`; `src/app/collab/room.ts` and `src/app/collab/session.ts` pass freeform/random room IDs into Trystero and IndexedDB; `src/app/document/io/source.ts` / `source-state.ts` track local file handle, Tauri path, download name, and saved version only. Contract must treat these as local/source metadata, not hosted identities.
- Auth alignment for Task 1: hosted modes must follow cookie-first browser sessions with bearer fallback for native/API callers, and reject cookie+bearer identity conflict rather than silently preferring one transport.
- Task 5 IO audit: local file behavior fans out through `createDocumentSourceActions` -> save/write/autosave plus reload watchers; the safe seam is a backend object injected into read/save/autosave orchestration while leaving File System Access, Tauri fs, and browser download helpers local-only.
- Task 5 hosted seam: app-side contract can cover hosted open/save/saveAs/autosave/metadata through an injected `HostedDocumentClient`, but must throw `hosted-runtime-unavailable` until Worker/API task slices provide the client.

- Task 2 package audit: `ch5-packages/packages/auth/elf-auth-client` and `firefly-cloud/packages/elf-auth-client` diverged on the public user identity field (`kiloUserId` vs `fireflyUserId`). The only active unscoped package consumer found in scope was `firefly-cloud/services/session-ingest`; other Firefly `fireflyUserId` references are internal worker-utils/service storage contracts, not this federation package boundary.
- Task 2 consumer migration note: sub-app consumers must import `@ch5me/elf-auth-client`, verify RS256/JWKS tokens with `createElfVerifier`, and read `payload.elfUserId`; internal Firefly DB/DO variables can keep `fireflyUserId`/`firefly_user_id` naming when not public auth payload.
- Task 7 schema audit: `api/` only had local Wrangler state before this slice, so the hosted document model needed a committed D1 migration plus TS schema/domain files rather than extending an existing Worker CRUD layer.
- Task 7 migration seam: local source state currently exposes handle/path/download/display metadata only; hosted D1 migration provenance should store source kind/name/fingerprint as metadata and never derive `document.id`, R2 keys, or ownership from those local fields.
- Task 3 naming audit: authoritative public auth docs still contain legacy `Kilo` surfaces in `ch5-company/elf-auth-cutover-spec.md`, `ch5-platform-package-catalog.md`, and federation telemetry docs. Treat those as rename-matrix inputs, not OpenPencil runtime concepts to rename in this slice.
- Task 3 compatibility lesson: canonical package schema should stay strict on `elfUserId`; backwards compatibility belongs at issuer/ingress/credential readers as one-way read-legacy/write-ELF adapters.

- Task 2 Durable Object export: wrangler requires Durable Object classes to be both a named module export AND importable with a local binding in the entrypoint. `export { X } from './mod'` alone fails at runtime with `ReferenceError: X is not defined`; must use `import { X } from './mod'; export { X }`.
- Task 2 binding names: `.ch5/services.yaml` binding names (`DB`, `DOCUMENTS`, `ASSETS`, `DOCUMENT_ROOM`) are the canonical contract — wrangler.jsonc bindings must match exactly for all stages.
